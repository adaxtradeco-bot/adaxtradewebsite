🎬 Hero Slider Enhancement - Image-Only Card Mode

Feature Status: ✅ Implementation Complete

📋 Implementation Summary

  • Schema Updated: Added useImageInsteadOfCard and cardImage fields to Slide interface
  • Component Enhanced: Created ImageCard component for image-only mode
  • Conditional Rendering: Slides now conditionally render card vs image based on useImageInsteadOfCard flag
  • Template Added: Hero-slider template added to section registry with new fields
  • Builder UI: Property panel updated with guidance for per-slide settings
  • Backward Compatible: All existing slides render exactly as before (default: useImageInsteadOfCard = false)

🔧 How to Use

In the page builder, when editing a hero-slider section:

  1. Open the JSON editor in the property panel
  2. For any slide, add: "useImageInsteadOfCard": true
  3. Add: "cardImage": "path/to/your/image.jpg"
  4. Save changes

📝 Example Slide Configuration

{
  "title": "Advanced Analytics",
  "subtitle": "Data-Driven Insights", 
  "description": "Harness the power of AI...",
  "tag": "AI Powered",
  "backgroundImage": "/assets/hero-bg-2.jpg",
  "useImageInsteadOfCard": true,
  "cardImage": "/assets/analytics-dashboard.jpg",
  "buttons": [...],
  "statistics": [...]
}

🎯 Key Features

  • Per-Slide Control: Each slide can independently use card or image mode
  • Mixed Slides: Slider supports combination of card slides and image slides
  • Responsive Design: Images use object-fit: cover and maintain aspect ratio
  • Theme Support: Images work with light/dark themes without contrast issues
  • Layout Preservation: Background, overlays, navigation remain unchanged
  • No Breaking Changes: Existing sliders continue to work without modification
