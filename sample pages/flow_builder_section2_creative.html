Notification Actions

Notify the right people at the right moment

Send email, SMS, fax, or internal notifications when a stage is entered, approved, rejected, completed, or overdue. Use template variables to personalize messages with live form data.

Trigger
Stage is entered, approved, rejected, or overdue
Channel
Email · SMS · Fax · Internal notification
Content
Template with live form field variables — name, amount, status, reason
Recipients
Submitter · Assignee · Manager · Any role or specific user
eg.

Leave request rejected → submitter receives SMS with rejection reason within 30 seconds. No manual follow-up. No forgotten notification.

Process Chaining

One workflow completion can start another automatically

When a stage completes or a final outcome is reached, trigger a downstream process and pass form data between parent and child. Clean separation of business domains.

Trigger point
Stage completion or final stage reached in parent process
Data mapping
Form fields from parent mapped to child process input — no re-entry
Child instance
Independent tracked instance created — linked to parent for audit
Chain depth
Multi-level chains supported — A → B → C → unlimited depth
eg.

Sales process closes → onboarding starts → on completion, billing setup starts. Three separate processes. Zero manual handoffs.

Scheduled Automation

Processes that start themselves — on a schedule

Create recurring workflows for weekly reviews, monthly compliance checks, or routine reminders. Managed entirely inside the platform — no cron jobs, no external scripts, no developer.

Frequency
Daily · Weekly · Monthly · Custom interval · Calendar-aware
Calendar logic
Skip weekends and public holidays — count only working days
Auto-assignment
Tasks assigned to roles automatically at start time
No dev required
Schedule configured visually inside the platform — no external scheduler
eg.

Every Monday at 8am, a weekly team checklist starts. Tasks assigned to each lead. Uncompleted items escalate to manager by Wednesday noon.

External System Actions

Push and pull data across any connected system

Fire REST calls or webhooks on stage events, send filtered payloads, and write API responses back into workflow fields. True bi-directional orchestration — not one-way export.

Method
REST POST · GET · PUT · PATCH to any HTTPS endpoint
Payload
Send filtered fields — not the full document — to external systems
Response mapping
Write API response values back into workflow form fields
Auth & retry
Bearer · API key · Basic auth — configurable retry policy on failure
eg.

IT request approved → API provisions AD account → response confirms provisioning → form field "Account Status" auto-updates to Active.

SLA & Escalation Engine

Missed deadlines escalate automatically

Define SLA timers on any stage and configure multi-level escalation chains. The engine reassigns, notifies managers, increases priority, or logs compliance events — without manual intervention.

Level 0 — Reminder
Notify current assignee before deadline — configurable threshold
Level 1 — Reassign
Move task to backup assignee or team lead automatically
Level 2 — Escalate
Notify manager + change task priority to critical
Level 3 — Compliance
Log SLA breach for audit + trigger corrective workflow automatically
eg.

IT ticket: reminder at 4h → escalate to lead at 8h → reach manager at 24h → SLA breach logged for compliance report.
